---

## Краткий обзор Black Duck Security Scan

GitHub Action для оркестрации инструментов безопасности (Polaris, Coverity, Black Duck SCA, SRM) через Bridge CLI. Ключевые архитектурные решения:

- **Bridge CLI** — промежуточный бинарник, загружаемый с Artifactory, с кэшированием и ARM-поддержкой
- **4 инструмента сканирования** — единый интерфейс через JSON-конфигурацию и `--stage --input`
- **Двойная загрузка SARIF** — как GitHub Artifact + в GitHub Advanced Security API
- **Air-gap режим** — работа без доступа к интернету
- **SSL/Proxy** — полноценная поддержка Enterprise-окружений
- **Factory-паттерн** — GitHub Cloud vs Enterprise Server

> [!NOTE]
> Полная архитектура задокументирована в `black-duck-security-scan-2.8.0/docs/Arch/ARCHITECTURE.md`

## Фазы развития CycloneDX Tauri UI

### Фаза 1 — Надёжность DAG Engine ✅ DONE

| # | Задача | Источник идеи | Приоритет | Сложность |
|---|--------|--------------|-----------|-----------|
| 1.1 | **RetryHelper для узлов DAG** — добавить retry с exponential backoff (15с→30с→60с) в `ExecutableNode::execute()` для отказоустойчивости внешних процессов (cdxgen, cyclonedx CLI) | `retry-helper.ts` | 🔴 Высокий | Низкая |
| 1.2 | **Агрегация ошибок валидации графа** — собирать все ошибки auto-wiring (MissingArtifact, TypeMismatch, CycleDetected) и показывать единым списком | `validators.ts` — агрегация | 🟡 Средний | Низкая |
| 1.3 | **Configurable build status** — настройка поведения NIST-узла: «блокировать при несоответствии» vs «только предупредить» (аналог `mark_build_status`) | `mark_build_status` паттерн | 🟡 Средний | Низкая |

**Результат фазы**: DAG Engine устойчив к сбоям, показывает все ошибки за раз, гибко реагирует на нарушения политик.
